# Diagrama de Implantação

## Introdução

O diagrama de implantação é um diagrama estrutural da UML que representa a organização física do sistema, mostrando como os componentes de software são distribuídos em elementos de hardware. Ele permite visualizar a infraestrutura necessária, como servidores, dispositivos e conexões de rede. Segundo IBM (2023), esse tipo de diagrama é fundamental para representar a topologia de execução do sistema e a distribuição de seus elementos.

Seu principal objetivo é destacar a relação entre os artefatos de software e os nós onde são executados, como servidores de aplicação e bancos de dados, contribuindo para a compreensão da arquitetura do sistema de um modo simplificada e fácil. 

## Metodologia

Para a elaboração do diagrama de implantação, foram adotadas boas práticas da UML e das referências utilizadas, considerando três aspectos principais: **nós, artefatos e conexões**.

Inicialmente, foram identificados os **nós do sistema**, que representam os elementos físicos ou ambientes de execução, tais como dispositivos como computadores, servidores e mobile, além de ambientes de execução como servidores web, containers ou máquinas virtuais.

Em seguida, foram definidos os **artefatos**, correspondentes aos componentes de software, como aplicações web, APIs, serviços backend e bancos de dados. Após isso, realizamos a alocação desses artefatos aos nós, indicando onde cada componente é executado.

Por fim, foram modeladas as **conexões entre os nós**, representando a comunicação entre os elementos do sistema por meio de protocolos como HTTP/HTTPS.

### Elementos do Diagrama de Implantação

Com base na metodologia, o diagrama do sistema *OrganizeSeuGrupo* é composto por:

- **Nós:** dispositivo do usuário, servidor de aplicação e servidor de banco de dados  
- **Artefatos:** frontend, backend e banco de dados  
- **Conexões:** comunicação entre cliente, servidor e banco via rede.
